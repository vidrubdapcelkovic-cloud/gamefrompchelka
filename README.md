# Прототип выживалки — MVP

Рабочий MVP 2D-игры с видом сверху на Phaser 3: фиксированная карта, добыча ресурсов, инвентарь, крафт, строительство стен и костров, здоровье, голод, слизни, ближняя и дальняя атака, а также ручное локальное сохранение. Основной результат — один автономный HTML-файл, который работает без интернета, Python, Node.js и сервера.

## Основной запуск: автономный файл

Готовая игра находится здесь:

```text
dist/survival-offline.html
```

## Главное меню и сохранения

При запуске сначала открывается главное меню с двумя независимыми слотами. В пустом слоте выберите «Новая игра», а в занятом — «Продолжить».

- `SAVE` или клавиша `K` сохраняет только выбранный слот.
- `LOAD` или клавиша `L` загружает только выбранный слот.
- `МЕНЮ` сохраняет активный слот и возвращает в главное меню после подтверждения.
- Перезапись занятого слота и сброс сохранения требуют подтверждения.
- Слот 1 и слот 2 хранят независимые состояния мира.

Старое валидное сохранение `survival-save-v1` один раз переносится в пустой слот 1. Автосохранения по таймеру нет.

Откройте этот файл двойным щелчком на компьютере или как обычный HTML-файл на Android. Подключение к интернету не требуется. Phaser, CSS и весь код игры уже встроены внутрь.

## Как перенести игру на Android

1. Скопируйте `dist/survival-offline.html` на телефон через USB, облачное хранилище, мессенджер или другой удобный способ.
2. Сохраните файл в доступную папку, например `Download` («Загрузки»).
3. Откройте файловый менеджер телефона, нажмите на `survival-offline.html` и выберите браузер.

Некоторые Android-браузеры по своим правилам безопасности не открывают локальные HTML-файлы из файлового менеджера или показывают их как текст. В таком случае попробуйте другой браузер, другой файловый менеджер с командой «Открыть с помощью» либо приложение, которое умеет открывать локальные HTML-файлы. Это ограничение конкретного браузера, а не зависимость игры от сервера.

На компьютере работают WASD и стрелки. На сенсорном экране используйте виртуальный джойстик в нижнем левом углу. Джойстик активируется только при касании внутри его внешнего круга; правая половина экрана остаётся свободной для будущих кнопок действий.

## Как заново собрать автономный файл

Node.js нужен только разработчику для сборки. На телефоне и после сборки он не требуется.

1. Откройте терминал в папке `gamefrompchelka`.
2. Выполните:

   ```text
   node build.js
   ```

3. Скрипт создаст или обновит `dist/survival-offline.html`.

Система сборки не использует npm и не скачивает файлы из интернета.

## Временный запуск для разработки

Этот способ удобен только при редактировании разделённых исходных файлов. Для готовой автономной версии он не нужен.

1. Откройте папку `gamefrompchelka` в Проводнике Windows.
2. Щёлкните по адресной строке Проводника, напишите `powershell` и нажмите Enter.
3. Выполните:

   ```text
   python -m http.server 8000
   ```

4. Откройте <http://localhost:8000>.
5. Для остановки сервера нажмите `Ctrl+C` в терминале.

Если команда `python` не найдена, попробуйте:

```text
py -m http.server 8000
```

## Управление

На компьютере:

- `W`, `A`, `S`, `D` или стрелки — движение;
- `1`–`5` — выбор быстрого слота;
- `E` (удерживать) — добыча выбранного объекта;
- `F` — использовать еду из активного быстрого слота;
- `I` — открыть или закрыть инвентарь;
- `C` — открыть или закрыть крафт;
- `B` — включить или выключить строительство;
- `Enter` — разместить выбранную постройку в режиме строительства;
- `Space` — атаковать;
- `K` — сохранить игру;
- `L` — загрузить сохранение;
- `Escape` — закрыть текущий режим: строительство, крафт или инвентарь.

На мобильном устройстве:

- джойстик слева — движение;
- `E` — удерживаемая добыча;
- `USE` — использовать еду;
- `ATTACK` — атаковать;
- `BUILD` — включить или выключить строительство;
- кнопки типов построек — выбрать стену или костёр;
- `PLACE` — разместить выбранную постройку; кнопка доступна только в режиме строительства;
- `CRAFT` — открыть или закрыть крафт;
- кнопка рюкзака — открыть или закрыть инвентарь;
- `SAVE` и `LOAD` — сохранить и загрузить игру;
- пять слотов снизу — выбрать активный предмет.

Персонаж движется с одинаковой скоростью во всех направлениях и не может выйти за границы мира.

## Мобильное масштабирование

Игра сохраняет внутреннее разрешение 960 × 540 и пропорции 16:9. Phaser использует `FIT` и `CENTER_BOTH`: Canvas целиком помещается в доступную область браузера, остаётся по центру и при необходимости оставляет свободные поля.

При изменении размера окна, повороте устройства, потере фокуса и возвращении из фона клавиатура и виртуальный джойстик сбрасываются. Интерфейс учитывает безопасные области экрана через `viewport-fit=cover` и CSS `env(safe-area-inset-*)`.

## Фиксированная карта

Карта 48 × 36 преобразуется из символов `G`, `S`, `W`, `R` в числовые индексы Phaser Tilemap. Для отображения используется одна программно созданная текстура из четырёх временных тайлов 32 × 32 и один слой поверхности. Внешние изображения пока не требуются. Вода с индексом `2` блокирует игрока, остальные поверхности проходимы. Размеры физического мира и камеры берутся из `WorldGrid`, а игрок появляется в центре стартовой клетки из `FixedMapData`.

## Фиксированные объекты мира

`FixedWorldObjects.js` содержит 14 деревьев, 10 камней и 8 ягодных кустов в формате `{ id, type, col, row }`. Размещение проверяется перед созданием объектов. Деревья и камни входят в одну статическую физическую группу и блокируют игрока; ягодные кусты остаются проходимыми. Временные текстуры создаются программно, а глубина объектов и игрока определяется их нижней точкой.

## Взаимодействие

`InteractionSystem.js` выбирает не более одной ближайшей цели в радиусе 72 пикселей. Для дерева используется центр блокировщика ствола, для камня и ягодного куста — центр изображения. Действие выполняется удержанием клавиши `E` или мобильной кнопки справа внизу. Прогресс показывается возле цели, а после завершения появляется временное сообщение. Длительность: дерево — 1500 мс, камень — 1700 мс, ягодный куст — 1000 мс.

Построенную деревянную стену можно разобрать тем же действием: подойдите к стене, повернитесь к ней и удерживайте `E` или мобильную кнопку действия 600 мс. Стена исчезнет, её клетка освободится, а на центре клетки появится возврат половины каждого ресурса из рецепта строительства с округлением вверх. Для текущей стены это `WOOD ×1`. Костёр не демонтируется: удержание действия возле него в течение 800 мс сначала пытается приготовить мясную похлёбку из `BERRIES ×2 + RAW_MEAT ×1`, а без доступного рецепта восстанавливает до 20 HP. Повторный отдых доступен через 30 секунд; готовка cooldown лечения не изменяет.

После завершения действия объект исчезает и оставляет один стак ресурса в своей клетке: дерево — `WOOD ×3`, камень — `STONE ×2`, ягодный куст — `BERRIES ×2`. Предмет автоматически подбирается в радиусе 28 пикселей, если в инвентаре есть место. Инвентарь содержит 25 слотов, включая 5 зарезервированных будущих быстрых слотов; каждый стак вмещает до 99 единиц. Наземные предметы не имеют физического тела и не блокируют персонажа. Временный HUD показывает общее количество ресурсов.

Первые 5 слотов постоянно показаны снизу и выбираются клавишами `1`–`5` или нажатием. Основная сетка 5×4 открывается клавишей `I` либо кнопкой рюкзака и закрывается повторным нажатием или `Escape`. Пока панель открыта, движение и взаимодействие отключены.

При открытом инвентаре стаки перемещаются двумя последовательными нажатиями: сначала выбирается занятый слот-источник, затем слот назначения. Одинаковые предметы объединяются до лимита 99, разные меняются местами, а нажатие на выбранный источник отменяет операцию. Разделение и выбрасывание стаков не реализованы.

Голод постепенно уменьшается, а при нулевом голоде персонаж теряет здоровье. Еда из активного быстрого слота используется клавишей `F` или мобильной кнопкой `USE`. Ягоды восстанавливают 20 голода. Мясная похлёбка восстанавливает до 45 голода и до 10 здоровья; при полных показателях она не расходуется. Сырое мясо употребить нельзя. После снижения здоровья до нуля управление отключается и появляется сообщение «Вы погибли»; возрождение не реализовано.

Панель крафта открывается клавишей `C` или мобильной кнопкой `CRAFT`. Доступны каменный топор (`WOOD ×3`, `STONE ×2`), каменная кирка (`WOOD ×2`, `STONE ×3`), каменный меч (`WOOD ×1`, `STONE ×2`) и лук (`WOOD ×3`, `SLIME_GEL ×1`). Ингредиенты списываются атомарно: при нехватке ресурсов или места инвентарь не изменяется. Панели крафта и инвентаря автоматически закрывают друг друга.

Инструмент ускоряет добычу только из выбранного быстрого слота: каменный топор сокращает время дерева до 750 мс, а каменная кирка — время камня до 850 мс. Неподходящий инструмент не даёт бонуса. Смена активного быстрого слота во время удержания отменяет действие без добычи и дропа.

Режим строительства включается клавишей `B` или кнопкой `BUILD`. В открывшемся списке выбирается деревянная стена (`WOOD ×2`) или костёр (`WOOD ×2 + STONE ×4`). Выбранная постройка размещается клавишей `Enter` или кнопкой `PLACE` в соседней клетке по последнему направлению движения. Красное превью означает запрещённую клетку, зелёное — допустимую. Обе постройки используют существующую статическую группу препятствий; поворот не реализован.

В мире находятся четыре слизня. Они замечают игрока поблизости, преследуют напрямую и наносят 5 единиц контактного урона. После попадания игрок получает 650 мс боевой неуязвимости и короткую вспышку; урон от голода при этом не блокируется. Если несколько слизней готовы атаковать в одном кадре, удар наносит только один из них.

Ближняя атака выполняется клавишей `Space` или кнопкой `ATTACK`, имеет радиус 52 пикселя, cooldown 450 мс и требует нового нажатия. Без оружия наносится 10 урона. Каменный меч в активном быстром слоте повышает урон до 15, не расходуется и не имеет прочности; меч в основном инвентаре или неактивном слоте бонуса не даёт. Попавший под удар слизень кратко вспыхивает; сообщение показывает фактически нанесённый урон и оставшееся здоровье. Рукой слизень погибает за три удара, мечом — за два, после чего один раз оставляет рядом два стака: `SLIME_GEL ×1–2` и `RAW_MEAT ×1`.

Ручное локальное сохранение выполняется клавишей `K` или кнопкой `SAVE`, загрузка — клавишей `L` или кнопкой `LOAD`. Обе команды работают только с активным слотом, выбранным в главном меню. Слоты независимы; старый ключ `survival-save-v1` используется только как источник одноразовой миграции. Автосохранение и автозагрузка отсутствуют; после смерти сохранить игру нельзя, но загрузить прежнее сохранение можно.

## Известные ограничения MVP

- Слизни преследуют игрока напрямую и не умеют прокладывать путь вокруг воды, деревьев, камней и построенных стен.
- На Android возможность открыть автономный локальный HTML зависит от браузера и файлового менеджера.
- Возрождение, автосохранение, звук, анимации, лут со слизней и респаун не реализованы.

## Структура проекта

- `index.html` — версия для разработки;
- `lib/phaser.min.js` — локальная библиотека Phaser 3.90.0;
- `src/main.js` — настройки Phaser и запуск игры;
- `src/VirtualJoystick.js` — виртуальный джойстик и обработка сенсорного ввода;
- `src/data/FixedMapData.js` — фиксированная карта 48 × 36 клеток и стартовая клетка игрока;
- `src/data/FixedWorldObjects.js` — фиксированное размещение деревьев, камней и ягодных кустов;
- `src/data/ItemCatalog.js` — каталог типов предметов и размеров стаков;
- `src/data/RecipeCatalog.js` — неизменяемый каталог рецептов;
- `src/data/BuildCatalog.js` — неизменяемый каталог построек и их стоимости;
- `src/data/CreatureCatalog.js` — параметры существ и контактного урона;
- `src/data/CombatConfig.js` — неизменяемые длительности боевой неуязвимости и вспышек;
- `src/world/WorldGrid.js` — независимые от Phaser координаты сетки и проверка проходимости;
- `src/systems/InteractionSystem.js` — выбор ближайшей цели для взаимодействия;
- `src/systems/HoldActionSystem.js` — прогресс, отмена и завершение удерживаемого действия;
- `src/systems/InventoryModel.js` — модель инвентаря из 25 слотов без зависимости от Phaser;
- `src/systems/PlayerStatsModel.js` — независимая модель здоровья и голода;
- `src/systems/CraftingModel.js` — проверка и выполнение рецептов через InventoryModel;
- `src/systems/BuildingSystem.js` — режим строительства, превью и runtime-стены;
- `src/systems/CreatureSystem.js` — runtime-существа, преследование, атаки и смерть;
- `src/systems/SaveSystem.js` — ручное локальное сохранение версии 1 через localStorage;
- `src/systems/GroundItemSystem.js` — runtime-стаки добытых предметов на земле;
- `src/ui/InventoryUI.js` — 5 быстрых слотов и скрываемая панель основного инвентаря;
- `src/ui/StatusHUD.js` — закреплённый HUD здоровья и голода;
- `src/ui/CraftingUI.js` — скрываемая панель выбора и запуска крафта;
- `src/GameScene.js` — мир, персонаж, управление, камера и интерфейс;
- `build.js` — создаёт автономную и многофайловую веб-версии;
- `dist/survival-offline.html` — готовая автономная игра.
- `docs/index.html` — готовая статическая страница для GitHub Pages;
- `docs/.nojekyll` — отключает обработку файлов через Jekyll.

## Публикация через GitHub Pages

Команда `node build.js` одновременно обновляет автономную версию в `dist` и версию для сайта в `docs`. Папку `docs` нужно сохранить в Git-репозитории вместе с исходниками.

После отправки проекта на GitHub откройте настройки репозитория: **Settings → Pages**. В разделе **Build and deployment** выберите **Deploy from a branch**, нужную ветку (обычно `main`) и папку `/docs`, затем сохраните настройки. Через некоторое время GitHub покажет адрес опубликованной игры.

Удалённый репозиторий и GitHub Pages настраиваются владельцем проекта; скрипт сборки не подключается к GitHub и ничего туда не отправляет.

### Структура ресурсов GitHub Pages

- `docs/assets/images` — отдельные изображения и фоны;
- `docs/assets/sprites` — персонажи, враги и спрайт-листы;
- `docs/assets/audio` — музыка и звуки;
- `docs/assets/maps` — карты и тайлсеты;
- `docs/assets/data` — JSON-файлы баланса и игровых данных.

Пока эти папки пусты и содержат только `.gitkeep`, чтобы Git сохранял их. Сборка не удаляет добавленные пользователем ресурсы.

Игровой код не выполняет `fetch` или `XMLHttpRequest` и не использует CDN, ES-модули или внешние игровые ресурсы. Встроенный полный дистрибутив Phaser содержит собственный универсальный код загрузчиков, но в этой игре он не вызывается: все данные уже находятся внутри HTML.
